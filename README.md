# Pull Request 리뷰하기

_병합하기 전에 다른 사람과 협업하여 작업을 다듬고 제안을 받아보세요._

## 환영합니다

모든 위대한 프로젝트는 협업에서 시작되며, 풀 리퀘스트(Pull Request)는 GitHub에서 팀워크의 기반입니다. 풀 리퀘스트를 통해 비동기적으로 함께 작업하고, 댓글 달기, 변경 요청, 승인 등을 통해 작업을 다듬을 수 있습니다.

- **대상**: 기술 개발자, 비기술 협업자, 학생, 관리자
- **배울 내용**:
  - 리뷰를 요청하고 담당자를 지정하는 방법
  - 다른 사람의 작업에 대한 리뷰를 제공하는 방법
  - 변경 사항을 제안하고 승인하는 방법
- **만들 것**: 간단한 게임의 풀 리퀘스트를 리뷰합니다.
- **사전 요구사항**: [GitHub 소개](https://github.com/skills-kr/introduction-to-github) 실습의 내용을 숙지하고 있어야 합니다.
  - 커밋(Commits)
  - 브랜치 및 풀 리퀘스트 생성
- **소요 시간**: 30분 이내

이 실습에서 다음을 수행합니다:

1. 풀 리퀘스트 열기
2. 본인을 담당자로 지정하기
3. 리뷰 남기기
4. 변경 사항 제안하기
5. 변경 사항 적용하기
6. 풀 리퀘스트 병합하기

### 이 실습을 시작하는 방법

실습을 내 계정으로 복사한 뒤, Octocat(Mona)이 첫 번째 레슨을 준비할 때까지 **약 20초** 기다린 후 **페이지를 새로고침**하세요.

실습을 복사할 때 다음 설정을 권장합니다:

- 소유자(owner)는 개인 계정 또는 조직(organization)을 선택하세요.
- 비공개 저장소는 [Actions 사용 시간이 소모](https://docs.github.com/en/billing/managing-billing-for-github-actions/about-billing-for-github-actions)되므로 공개 저장소를 만드는 것을 권장합니다.

20초 후에도 실습이 준비되지 않으면:

1. 새 저장소가 생성된 후 약 20초 기다린 다음 페이지를 새로고침하세요.
2. 저장소에 생성된 이슈의 단계별 안내를 따르세요.
3. 페이지가 자동으로 새로고침되지 않으면 [Actions](../../actions) 탭을 확인하세요.
   - 작업이 실행 중인지 확인하세요. 때로는 조금 더 오래 걸릴 수 있습니다.
   - 실패한 작업이 표시되면 이슈를 제출해 주세요. 버그를 발견하셨네요! 🐛
